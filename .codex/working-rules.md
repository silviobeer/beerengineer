# Codex Working Rules

- Autonom arbeiten ist der Standard.
- Keine Rueckfragen fuer normale technische Entscheidungen, kleinere Architekturentscheidungen, Refactorings, Tests, Doku oder klar priorisierte Review-Fixes.
- Rueckfragen nur bei:
  - echter fachlicher Unklarheit
  - destruktiven Schritten
  - externen Auswirkungen
  - Scope-Aenderungen mit realen Produktentscheidungen
- Formale Freigaben wegen Tooling oder Sandbox moeglichst ohne zusaetzliche inhaltliche Schleifen behandeln.
