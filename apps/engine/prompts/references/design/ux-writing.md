# UX Writing

Words are part of the interface system, not filler between layout decisions.

- Prefer verbs that describe the outcome the user is choosing, not generic labels that require guesswork.
- Empty states should state what is missing, why it matters, and what the next action is.
- Error copy should be specific enough to guide recovery without exposing internal stack language.
- Navigation labels should stay short and scannable, especially in sidebars and top-level tabs.
- If the audience is operational, terse language can work; if the audience is broader, explain more and sound less internal.
