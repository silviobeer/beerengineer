# Color And Contrast

Color should separate roles, states, and emphasis without making the interface muddy.

- Build around a small set of intentional surfaces. Too many near-adjacent grays and tints make the UI feel foggy.
- Give text colors enough contrast against every declared surface, especially muted labels, placeholder text, and secondary metadata.
- Use accent colors sparingly enough that they still mean something. If everything is accented, nothing is.
- State colors should survive beside each other in a dense board or table. Success, warning, and error must remain distinguishable without looking loud.
- When the brief allows, warmer or earthier neutrals often feel more specific than the default cool gray stack.
- Dark mode should be deliberate if included; do not invert values mechanically and call it finished.
