# Responsive Design

Responsive checks are part of QA because many otherwise polished interfaces fail at the last compression step.

- Reorder content by user priority, not by what happens to fit after shrinking.
- Preserve the primary action, current context, and status information at narrow widths.
- Tables, boards, and multi-column layouts need an intentional fallback pattern rather than simple overflow and hope.
- Spacing and typography can tighten on mobile, but not so far that touch comfort and readability collapse.
- Empty, loading, and error states still need to read as designed screens on small viewports.
