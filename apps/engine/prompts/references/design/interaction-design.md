# Interaction Design

Interaction polish should survive keyboard and pointer use, fast paths and mistakes.

- Primary actions should be obvious from placement, contrast, and wording before a tooltip is needed.
- Keep destructive or high-risk actions visually distinct from routine ones without hiding them.
- Dense screens still need comfortable hit targets and clear focus states.
- Hover affordances can enrich a desktop flow, but critical actions must also be visible and reachable without hover.
- Inline validation and error states should explain the problem in plain language close to the control that caused it.
- Do not force the user to decode iconography when a short label would remove doubt.
